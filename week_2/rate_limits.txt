Batch size calculation:
e.g.
model = gemini-2.5-flash-lite

req per minute -> 10
req per second -> 10/60 = 0.167
1 req -> 1/0.167 = 6s (time for each api call)

if req per day = 20,
- batch_size = 1 -> 8 rows = 8 api calls
- batch_size = 2 -> 8 rows = 4 api calls
- batch_size = 4 -> 8 rows = 2 api calls

min_batch_size = ceil(total_rows/rpd)
               = ceil(100/20)
               = 5
    hence, must process at least 5 rows per api call to fit 100 rows within 20 daily requests
